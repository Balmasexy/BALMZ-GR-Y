# Contributing to BALMZ Token

Thank you for your interest in contributing to the BALMZ Token project! We welcome contributions from the community.

## How to Contribute

### 1. Fork the Repository
Fork the [BALMZ-GR-Y](https://github.com/Balmasexy/BALMZ-GR-Y) repository to your GitHub account.

### 2. Clone Your Fork
```bash
git clone https://github.com/YOUR_USERNAME/BALMZ-GR-Y.git
cd BALMZ-GR-Y
```

### 3. Create a Feature Branch
```bash
git checkout -b feature/your-feature-name
```

### 4. Install Dependencies
```bash
npm install
```

### 5. Make Your Changes
- Follow the existing code style and conventions
- Add tests for new functionality
- Update documentation as needed

### 6. Run Tests
```bash
npm run test
npm run test:coverage
```

### 7. Compile Smart Contracts
```bash
npm run compile
```

### 8. Commit Your Changes
```bash
git add .
git commit -m "Description of your changes"
```

### 9. Push to Your Fork
```bash
git push origin feature/your-feature-name
```

### 10. Create a Pull Request
Open a pull request to the main repository with a clear description of your changes.

## Code Style Guidelines

- **Solidity**: Follow [Solidity Style Guide](https://docs.soliditylang.org/en/latest/style-guide.html)
- **JavaScript**: Use consistent indentation (2 spaces)
- **Comments**: Add meaningful comments for complex logic
- **Tests**: Write comprehensive tests for all smart contract functions

## Testing Requirements

Before submitting a PR, ensure:
- All tests pass: `npm run test`
- Code coverage is adequate: `npm run test:coverage`
- Smart contracts compile without warnings: `npm run compile`

## Reporting Issues

If you find a bug or have a feature request:
1. Check existing [GitHub Issues](https://github.com/Balmasexy/BALMZ-GR-Y/issues)
2. Create a new issue with clear description and steps to reproduce
3. Include relevant details like environment, version, and error messages
